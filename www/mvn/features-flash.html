This feature is the flash concept taken from Ruby On Rails.

In JSF, the flash is exposed naturally via the new Unified Expression Language in Java EE 5. It is implemented via a custom ELResolver that introduces a new implicit object called "flash".

Using the flash is simple, and semantically identical to the way it works in Rails. It's a Map. Stuff you put in the Map will be accessible on the "next" view shown to user. The Map will be cleared when the user has been shown the "next" view.

Put fooValue in the flash under key foo using c:set. Note that things stored in the flash during this request are only retrievable on the next request. If you want to store something on this request and see it on this one as well, use either #{flash.now.foo} or #{requestScope.foo}. The former is simply an alias for the latter. <c:set var="#{flash.foo}" value="fooValue" />

Going to the next page, we have:

Value of the previous request's foo: <h:outputText value="#{flash.foo}" /> fooValue
Put barValue in the flash.now under key bar. <c:set var="#{flash.now.bar}" value="barValue" />
Value of #{flash.now.bar}, should be barValue. barValue

Going to the next page, we have:

Value of the previous request's foo: <h:outputText value="#{flash.foo}" />
Value of the this request's bar. Should be null: <h:outputText value="#{flash.bar}" />
Put banzai in the flash.now under key buckaroo. <c:set var="#{flash.now.buckaroo}" value="banzai" />
Value of #{flash.now.buckaroo}, should be banzai. banzai
Promote buckaroo to stick around for the next request. <c:set var="#{flash.now.buckaroo}" value="#{flash.keep.buckaroo}" />

Going another page further, we have:

Show that buckaroo is still here. <h:outputText value="#{flash.buckaroo}" /> banzai
