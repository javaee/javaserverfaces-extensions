An overview of the JSF Avatar System

Outline


I. Introduction

Basic idea: Allow JSF to continue to be a pseudo-disconnected controller
for the AJAX portion of a web application.

Why this is important: JSF's strengths are its component abstraction and
event model.  Why re-invent something specifically for AJAX when we can
fully leverage the JSF model while preserving the AJAXian way?
